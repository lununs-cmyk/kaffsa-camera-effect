# SOOP Camera

브라우저에서 실행되는 단일 페이지 카메라 연출 프로젝트입니다.

## 실행 방법

별도의 빌드 과정이 없습니다.

1. 저장소를 내려받습니다.
2. `index.html`을 브라우저에서 엽니다.
3. GitHub Pages를 사용하는 경우 저장소 설정에서 Pages 배포 소스를 지정합니다.

## 사진 에셋 추가

현재 기본 이미지 주소는 모두 비어 있습니다. 따라서 처음 실행하면 사용 가능한 사진 에셋이 없다는 안내가 표시됩니다.

`index.html` 상단의 `CONFIG.MODES`에서 각 모드의 `images` 배열에 로컬 상대 경로나 외부 URL을 넣습니다.

```javascript
{
  id: "slow-motion",
  name: "슬로 모션",
  text: "1",
  images: [
    "assets/images/slow-motion-01.jpg",
    "https://example.com/slow-motion-02.jpg"
  ]
}
```

로컬 이미지와 외부 이미지를 같은 배열에서 함께 사용할 수 있습니다.

사진 배열이 비어 있거나 사진을 불러오지 못하면 해당 모드 번호는 화면에 표시되지 않습니다. 모든 모드가 비어 있으면 촬영 버튼이 비활성화되고 에셋 안내 문구가 표시됩니다.

## 기본 모드 순서

1. 슬로 모션
2. 비디오
3. 사진
4. 인물 사진
5. 파노라마

초기 선택 모드는 1번 슬로 모션입니다.

## 조작

- 촬영 버튼: 촬영 효과 실행
- 좌우 화살표: 촬영 후 사진 넘기기
- 모바일 스와이프: 촬영 후 사진 넘기기
- 키보드 좌우 방향키: 촬영 후 사진 넘기기
- Enter 또는 Space: 촬영
- Escape: 다시 촬영
- 사진 저장: 현재 보고 있는 사진 저장

## 참고

외부 이미지 서버가 CORS 다운로드를 차단할 경우 사진 저장 버튼은 원본 이미지를 새 창으로 엽니다.
로컬 에셋을 사용할 때는 `assets/images` 폴더에 파일을 넣고 상대 경로로 연결하는 방식을 권장합니다.
